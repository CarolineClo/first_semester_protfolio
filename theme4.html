A website with a game

Before creating the game we learned about game mechanics, and the restrictions of this theme. I chose to make a game about potty training as I imagined it would have fun design elements and (as objects tend to fall into a toilet) simple falling mechanics.

I created the game for parents to play together with their toddlers as a way to start a conversation abut potty training. By clicking on the falling objects, the player can save certain items from falling into the toilet. The teddy bear, phone and sock do not belong in the toilet so click them and receive a point or lose a point if they fall into the toilet. The pee and poop should be allowed to fall into the toilet, so if the player clicks on them they will lose a life. To win the player must have ten points when the timer runs out. But if the player loses all three lives at anytime, they lose the game.

In this assignment we learned css positioning which lets us take an element outside of the normal flow of a document which we learned in basic web. The game had to have various screens stacked on top of each other within the game scene so I used position absolute to achieve this. I also absolute positioning on all the game sprite elements in the game so I control where they should sit within the game.

We were introduced to the transform and transition properties in css. The transform property allows us to change an elements appearance by scaling, rotating or skewing and the transition property defines the transition of these different states.

We learned how to apply these properties by creating a keyframe sequence. I used keyframe animations throughout my game. For example the falling objects I mentioned earlier are animated using the sequence:

@keyframes falling
{ 0% {transform:translateY(-10%)}
100% {transform: translateY(380%)} }

In this assignment we were also introduced to javascript. When an html document is linked to a javascript, any element can be accessed using the function queryselector and we can then affect that element within javascript, such as adding or removing a class from an element. We learned that we can add events to the document by using eventlistener injavescript. An example of how I implemented this was to animate my falling objects after the player clicks on them. I instructed my falling objects to listen for the event “click” and then perform another keyframe animation, which was to spin and disappear.

A crucial element to the game is the points counter. We learned how to solve this using variables, which is a name that we tell javscript is a value. In the case of my game the variable is playerPoints=0 at the start of my game. By adding the instruction playerlives++ to the function that happens after a falling object is clicked, the player points will increase by one point every click.

Another element crucial to the game is the timer. I decided to illustrate the passing of time with keyframe animation of a shrinking bar. I decided this was best as the game is aimed at young kids who would understand that better than numbers counting down.

Asset List and Documentation

As the game is aimed at children I was attracted to round, friendly shapes and bright pastel colours for the assets. I also liked minimal shading to add a bit of depth to the characters.

In this theme we learned how to transform rough sketches into high fidelity characters in Adobe illustrator, we learned that characters look more alive if they are asymmetrical so I applied this to all characters.

We learned about the importance of UI within a game. I created extra shading on clickable assets so they can be set apart from non clickable elements within the game.
